# 人体红外热成像伪彩渲染系统

本项目是一个用于展示和增强人体红外热成像数据的网页系统。系统可以读取人体红外热成像图片、温度矩阵和`Result.xml`检测结果文件，然后在网页中使用Canvas重新绘制伪彩图，并通过右侧滑块实时调整温标、对比度、锐化和CLAHE局部对比度增强参数，让人体温区边缘显示得更接近热成像相机软件的效果。

项目当前版本已经去掉“人体区域Mask白底显示”功能，主要增强方式为：

```text
温度矩阵
→ 自动温标计算
→ 非锐化掩蔽边缘增强
→ CLAHE局部对比度增强
→ Gamma非线性调整
→ 伪彩色带映射
→ Canvas实时重绘
→ ROI点位叠加显示
```

---

## 1. 项目能做什么

本系统的核心目标不是简单显示一张JPG图片，而是根据真实温度矩阵重新生成热成像伪彩图。

用户可以在网页中完成以下操作：

1. 查看五张人体红外热成像图片。
2. 对有温度矩阵的图片进行Canvas伪彩重绘。
3. 拖动右侧滑块，实时改变伪彩图显示效果。
4. 调整温位、温宽、Gamma、锐化程度和CLAHE参数。
5. 切换不同伪彩色带，例如分级热成像、铁红、灰度和彩虹色。
6. 显示`Result.xml`中的ROI点位，例如右腋、左腋、右掌、左掌、督脉等。
7. 对没有温度矩阵的图片，自动退化为原始JPG预览。

通俗理解：

```text
普通JPG图片：只能看，不能根据温度重新变色。
温度矩阵：每个像素都有真实温度，可以重新生成不同颜色的热图。
本系统：把温度矩阵变成可以交互调节的网页热成像图。
```

## 4. 数据文件说明

### 4.1 JPG伪彩图片

JPG图片是热成像设备或相机软件导出的伪彩显示图。它主要用于页面预览和图片切换。

需要注意：JPG图片本身不等于真实温度数据。JPG只保存颜色，不保存每个点的真实温度。

### 4.2 TXT温度矩阵

TXT文件保存的是温度矩阵，也就是每个像素点对应的温度值。

例如：

```text
34.34 34.36 34.38 34.38 34.42 ...
34.35 34.41 34.39 34.39 34.41 ...
```

本项目中图像矩阵尺寸为：

```text
宽度：460
高度：620
总点数：460 × 620 = 285200
```

所以一个完整的TXT温度矩阵中应该有`285200`个温度值。

后端会读取这些温度值，然后返回给前端。前端再通过Canvas把这些温度值重新绘制成伪彩图。

### 4.3 Result.xml

`Result.xml`用于保存检测相关信息，主要包括：

1. 图像宽度和高度。
2. 每张图的默认温标信息。
3. ROI点位信息。
4. 用户、年龄、性别、医生等基本信息。

ROI是“感兴趣区域”的意思，英文全称为Region of Interest。比如右腋、左腋、右掌、左掌、督脉等位置，都可以看作ROI。

```text
Python 3.9及以上
```

```bash
cd thermal_web_demo/backend
pip install -r requirements.txt
```

`requirements.txt`中主要依赖包括：

```text
fastapi
uvicorn
numpy
```

前端使用Node.js、Vue3和Vite。

如果使用较新的Vite版本，建议Node版本为：

```text
Node.js 20.19及以上，或Node.js 22.12及以上
```

如果你的电脑是Node.js 18，可以把`package.json`中的版本固定为下面这样：

```json
{
```

## 8. 前端页面功能说明

### 8.1 左侧图片列表

页面左侧或下方会显示五张热成像图片列表。点击某张图片后，前端会请求后端对应接口。

如果该图片有同名温度矩阵，页面会进行Canvas伪彩重绘。

如果该图片没有温度矩阵，页面会显示原始JPG。

### 8.2 主显示区

主显示区使用Canvas绘图。

Canvas不是简单放图片，而是根据温度矩阵逐点计算颜色。

绘制过程可以理解为：

```text
读取温度值
→ 判断这个温度在当前温标范围中的位置
→ 转换成0到1之间的比例
→ 根据比例选择伪彩颜色
→ 把颜色画到Canvas对应像素点上
```

### 8.3 右侧控制区

右侧控制区用于调节伪彩图显示效果。

包括：

```text
温位
温宽
显示最低温
显示最高温
Gamma对比度
锐化程度
CLAHE局部对比度增强
色带选择
ROI显示开关
```

## 9. 核心参数解释

### 9.1 温位

温位可以理解为当前显示温标的中心温度。

例如：

```text
显示最低温：34℃
显示最高温：40℃
温位 = (34 + 40) / 2 = 37℃
```

温位调高，整体显示会偏向高温范围；温位调低，整体显示会偏向低温范围。

### 9.2 温宽

温宽指显示温标覆盖的温度范围。

例如：

```text
显示最低温：34℃
显示最高温：40℃
温宽 = 40 - 34 = 6℃
```

温宽越小，颜色变化越明显，边缘更清楚，但也更容易过度增强。

温宽越大，颜色变化越平滑，画面更稳定，但细节可能不明显。

### 9.3 显示最低温和显示最高温

显示最低温和显示最高温是当前颜色映射范围的上下限。

低于显示最低温的点会被映射为最低颜色。

高于显示最高温的点会被映射为最高颜色。

如果显示最高温设置得太低，整张图可能会变成一整片高温颜色。

如果显示最低温设置得太高，低温区域会丢失细节。

### 9.4 Gamma

Gamma是一种非线性对比度调整方法。

通俗理解：Gamma可以改变“中间温度区域”的明暗和颜色分布。

```text
Gamma < 1：中高温区域更亮，更容易突出人体热区。
Gamma = 1：保持线性显示。
Gamma > 1：低温区域层次更多，但高温区域可能不够突出。
```

本项目建议默认值：

```text
Gamma = 0.95
```

### 9.5 锐化程度

锐化程度用于增强温度变化明显的位置，也就是边缘区域。

本项目使用的是非锐化掩蔽算法，核心思想是：

```text
增强图 = 原始温度 + 锐化系数 × (原始温度 - 模糊后的温度)
```

通俗理解：

```text
如果某个点和周围差异很大，说明这里可能是边缘。
算法会把这种差异再放大一点，让边缘看起来更清楚。
```

建议范围：

```text
0.15 ~ 0.35
```

不建议调得太高，否则会产生假边缘和噪点。

### 9.6 CLAHE局部对比度增强

CLAHE的英文全称是Contrast Limited Adaptive Histogram Equalization，中文一般叫“限制对比度自适应直方图均衡化”。

名字比较长，可以简单理解为：

```text
把整张图分成很多小块，分别增强每个小块里的温差细节，
同时限制增强强度，避免噪声被无限放大。
```

普通全局增强只看整张图，容易被极端高温或低温点影响。

CLAHE看的是局部区域，所以更适合增强人体热成像中的局部细节，比如腋下、手掌、膝盖、躯干边缘等。

本项目中CLAHE有三个参数：

| 参数 | 含义 | 推荐范围 |
|---|---|---|
| CLAHE增强强度 | 控制CLAHE结果和原始结果混合比例 | 0.40 ~ 0.75 |
| CLAHE裁剪阈值 | 限制局部对比度增强的强度 | 2.0 ~ 3.0 |
| CLAHE局部块大小 | 每个局部增强区域的大小 | 40px ~ 56px |

当前推荐默认值：

```text
CLAHE增强强度：0.60
CLAHE裁剪阈值：2.5
CLAHE局部块大小：48px
```

## 10. 为什么相机软件的边缘更明显

直接根据温度矩阵画图时，通常是线性映射：

```text
温度越高，颜色越接近高温颜色；
温度越低，颜色越接近低温颜色。
```

这样做虽然真实，但颜色变化会比较平滑，边缘不明显。

商业热成像软件通常会加入多种显示增强算法：

```text
温度矩阵
→ 去噪
→ 自动温标
→ 局部对比度增强
→ 边缘锐化
→ 非线性伪彩映射
→ 插值放大
→ 显示
```

所以商业软件中的边缘明显，不一定代表真实温度突然断开，而是经过视觉增强后的显示效果。

本项目目前已经实现的增强方法包括：

| 方法 | 是否实现 | 作用 |
|---|---|---|
| 自动温标 | 已实现 | 使用P2~P98分位数自动确定默认显示范围 |
| 伪彩映射 | 已实现 | 把温度值映射成颜色 |
| Canvas实时重绘 | 已实现 | 滑块变化后立即重新渲染 |
| Gamma增强 | 已实现 | 调整中间温度区域显示效果 |
| 非锐化掩蔽 | 已实现 | 增强温区边缘 |
| CLAHE局部对比度增强 | 已实现 | 增强局部小温差细节 |
| 人体Mask | 已删除 | 当前版本不再使用 |

## 12. 重要说明：显示增强不等于真实测温

本项目中的Gamma、锐化、CLAHE都属于显示增强。

它们的作用是让人眼更容易观察温区边缘和局部差异。

但是不要把增强后的结果当成真实温度矩阵使用。

正确做法是：

```text
真实测温、诊断判断、数据统计：使用原始温度矩阵。
网页显示、辅助观察、边缘增强：使用增强后的显示结果。
```

也就是说，增强图只是“看得更清楚”，不是“温度真的发生了改变”。

## 13. 常见问题和解决方法

### 13.1 后端报错：Could not import module "main"

原因：运行`uvicorn main:app --reload`时目录不对。

错误目录可能是：

```text
thermal_web_demo
```

正确目录应该是：

```text
thermal_web_demo/backend
```

解决方法：

```bash
cd thermal_web_demo/backend
uvicorn main:app --reload
```

### 13.2 前端提示Node版本过低

如果出现：

```text
Vite requires Node.js version 20.19+ or 22.12+
```

有两种解决方式。

方式一：升级Node.js到20或22。

方式二：固定Vite版本为`5.4.11`，适配Node.js 18。

### 13.3 前端提示`.vue`文件解析失败

原因：没有配置Vue插件。

解决方法：检查`vite.config.js`，确保启用了：

```js
plugins: [vue()]
```

### 13.4 图片显示原图，不能动态变色

原因：该图片没有对应的同名TXT温度矩阵。

例如有：

```text
2024-07-01-08-30-43.jpg
```

但没有：

```text
2024-07-01-08-30-43.txt
```

解决方法：补充同名温度矩阵文件。

### 13.5 一打开画面变成一整块颜色

常见原因是温标范围设置不合适。

例如真实温度是34℃到42℃，但显示最高温只有33℃，那么所有像素都会被当成高温，画面就会变成一整块高温颜色。

本项目默认使用P2~P98分位数自动计算温标，可以减少这种情况。
