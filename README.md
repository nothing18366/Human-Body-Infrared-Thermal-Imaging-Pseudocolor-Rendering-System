# 人体红外热成像伪彩渲染系统 V3.0 医疗增强版（背景替换）

## 💡 一句话概述

本项目将 **原始温度矩阵** 通过 **双边滤波、CLAHE、Sobel、等温区分层等一系列医疗级增强算法**，在浏览器端实时重绘为可交互的伪彩医学热图，效果对标专业热成像软件。

- ✅ **双边滤波** – 保边去噪，平滑图像的同时保留温区边缘
- ✅ **Sobel 梯度增强** – 突出体温边界，让病灶轮廓更清晰

- ✅ **背景替换 V3.0** – 基于Otsu阈值分割自动识别人体轮廓，替换背景为白色，边缘平滑过渡

  `温度矩阵 → Bilateral → CLAHE → Sobel → USM → Normalize → Isotherm → Gamma → Medical Palette → Canvas`

## 🔬 医疗热成像显示原理

红外热成像设备采集到的是一幅二维温度分布（每个像素对应一个温度值）。  
为了让人眼观察，需要将温度映射为可见光颜色，这个过程称为"伪彩渲染"。

本系统模拟了主流热像仪的处理流程：

1. **去噪** – 运用双边滤波，在平滑图像的同时保留人体边界，避免边缘模糊。
2. **局部对比度增强** – CLAHE 把图像分成小块，各自均衡直方图，使腋下、掌心等微小温差更明显。
3. **边缘增强** – Sobel 算子计算温度梯度，USM 将梯度信息加回到原图，使温区轮廓更加锐利。
4. **等温分层** – 将连续温度量化为若干等级，便于医生快速判断温度分布梯度。
5. **医疗色带** – 采用黑→蓝→绿→黄→红→白的专用色带，符合临床习惯，低温区冷色调，高温区暖色调。
6. **自动温标** – 使用 P2~P98 分位数自动确定显示窗口，避免个别极端值拉宽色阶。

## 🧠 关键算法原理

### 1. 双边滤波（Bilateral Filter）

- **用途**：去噪同时保护边缘。
- **原理**：每个像素的新值由邻域像素加权平均得到，权重由 **空间距离** 和 **温度差异** 共同决定。  
  空间上近的、温度相似的像素权重大；相反权重小。这使得平坦区域平滑，边缘处则保持原样。

### 2. CLAHE（限制对比度自适应直方图均衡化）

- **用途**：增强局部细节，克服全局直方图均衡化"过曝"的问题。
- **原理**：将图像分成不重叠的瓦片，对每个瓦片单独做直方图均衡；均衡前对直方图进行裁剪（clip limit），把超过阈值的频数均匀分给所有灰度级，避免噪声放大。最后通过双线性插值合并各瓦片结果。

### 3. Sobel 算子（Sobel Gradient）

- **用途**：检测温度变化剧烈的边缘区域。
- **原理**：使用 3×3 的水平、垂直卷积核，分别计算 x、y 方向的温度梯度，然后求幅值。梯度幅值高的地方即可能是器官边界或异常热区。

### 4. USM（非锐化掩蔽）

- **用途**：一种经典的图像锐化技术。
- **原理**：`锐化图 = 原图 + amount × (原图 - 高斯模糊图)`。括号内的部分相当于提取了高频边缘信息，再将这部分信息放大叠加回去，使边缘更突出。

### 5. 等温区分层（Isotherm）

- **用途**：将连续温度离散化为多个温度台阶，每一台阶内颜色相同，相邻台阶颜色跳跃，形成"等高线"效果。
- **原理**：将归一化值 `v` 量化为 N 个台阶：`v' = (floor(v × N) + 0.5) / N`，然后用该值映射色带。台阶数量越多，过渡越平滑；越少，等温线效果越明显。

---

### 6. 背景分割与替换（Background Replacement V3.0）

- **用途**：将人体红外热图与背景分离，替换为白色背景，使身体轮廓更加清晰。
- **原理**：
  1. **Otsu 阈值分割** – 对原始温度矩阵构建 256-bin 直方图，计算最大化类间方差的阈值，自动区别人体（高温）与背景（低温）。
  2. **形态学清理** – 闭运算（先膨胀后腐蚀）填充人体区域内的空洞，开运算（先腐蚀后膨胀）去除背景中孤立噪点并扩展身体区域。
  3. **连通域分析** – 两遍扫描（Two-Pass）算法 + Union-Find 并查集，标记所有连通区域，仅保留最大的连通域（人体），消除误分割碎片。
  4. **可分离高斯模糊** – 对二值掩码分别做水平和垂直 1D 高斯卷积，生成 0~1 之间的平滑 alpha 值，O(2n·r) 替代 O(n·r²)。
  5. **Alpha 混合** – 在伪彩绘制阶段，每个像素按 `rgb_body × alpha + white × (1−alpha)` 与白色背景线性混合。

## 🚀 生产环境部署说明

### 1. 后端部署

```bash
cd thermal_web_demo/backend
pip install -r requirements.txt
uvicorn main:app --reload      #运行
```

生产环境建议使用 gunicorn + uvicorn workers：

```bash
gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
```

### 2. 前端构建与部署

```bash
cd thermal_web_demo/frontend
npm install
npm run build   # 生成 dist 目录
npm run dev     #运行
```

将 `dist/` 目录部署到任意静态服务器（Nginx、Caddy、OSS 等），并配置反向代理使 `/api` 指向后端。

### 3. 数据文件准备

将 `Result.xml`、JPG 图片、同名 TXT 温度矩阵放入 `backend/data/` 目录。  
确保文件名严格一致，例如 `xxx.jpg` 对应 `xxx.txt`。  
TXT 文件内容为 **空格** 或 **逗号** 分割的温度值，数量 = 宽 × 高。

### 4. 离线/内网运行

后端与前端均可完全离线运行，无需外部 CDN。  
所有依赖（FastAPI、Vue、Vite）均可在本地安装，无互联网依赖。

## ❓ 常见问题

**Q: 为什么有些图像只能显示原图，无法动态调节？**

A: 因为缺少对应的同名 `.txt` 温度矩阵。请补充温度矩阵文件。

**Q: 调节参数后画面反应慢？**

A: 双边滤波等算法涉及大量浮点运算。可在 UI 中降低空间 σ 或关闭不必要的增强。现代电脑上 460×620 的图像通常在 100 ms 内完成。

**Q: 增强后的图能否直接用于诊断？**

A: 增强图仅用于辅助观察，定量测量、诊断依据请使用原始温度矩阵。

**Q: 如何添加更多色带？**

A: 在 `App.vue` 的 `pseudoColor` 函数中增加 `if (name === 'yourname')` 分支，并在 `paletteCss` 中添加对应 CSS 渐变即可。
